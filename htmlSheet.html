Basic Tags

<html> </html>

Creates an HTML document.

<head> </head>

Sets off the title & other info that isn't displayed.

<body> </body>

Sets off the visible portion of the document.

<title> </title>

Puts name of the document in the title bar;
bookmarking pages, this is what is bookmarked.

All the basic HTML tags can be described as structural elements that form the foundation of an HTML document.
Together, they:

  • Define the structure and layout of a webpage.
  • Provide a clear hierarchy for content (e.g., <html> as the root, <head> for metadata, and <body> for visible content).
  • Act as the building blocks of any webpage, ensuring browsers understand how to render the document correctly.
  • In essence, these tags establish the framework for organizing and presenting content on the web.


Formatting

<p>

Defines a paragraph of text.

<br>

Inserts a line break.

<div> </div>

Used to format block content with CSS.
Defines a container or division in the document.

<span> </span>

Formatting tags in HTML play a crucial role in structuring and styling content so it can be clean, modular, and visually appealing. They help define how content should appear on a webpage and allow for better organization of the layout.
Here's an in-depth explanation of the formatting tags:

Paragraph Tag (<p>)

  • Defines a paragraph of text.
  • Adds vertical spacing before and after the paragraph to separate content.
  • Groups sentences or blocks of text into distinct sections.
  • Useful for displaying long bodies of text, articles, blog posts, or any written content that needs proper spacing.

Line Break Tag (<br>)

  • Inserts a single line break.
  • Forces content following it to appear on the next line.
  • It is an empty tag (no closing tag required).
  • Ideal for breaking lines in places where paragraphs are not required, such as addresses, poetry, or form content.

Division Tag (<div>)

  • Acts as a "box" to wrap multiple elements together.
  • Has no inherent styling but is often used with CSS to apply layout, spacing, or design.
  • Widely used for: Structuring layout components (e.g., headers, sidebars, or footers).

Inline Container Tag (<span>)

  • Defines an inline container for small sections of text.
  • Similar to <div>, but inline, meaning it does not break into a new line.
  • Typically used for applying CSS styles or JavaScript to a specific portion of text.

Text tags in HTML are used to define the hierarchy, emphasis, and meaning of text content. They improve both the visual presentation and the semantic meaning of the content, making it easier for browsers, search engines, and assistive technologies to interpret the information.
Here's a detailed explanation of the Text Tags:

Heading Tags(<h1> to <h6>)

  • Create headings of varying levels, where <h1> is the largest and <h6> is the smallest.
  • Headings convey a hierarchy for your content.
  • Search engines use headings to understand the structure and importance of your content.
  • Visually, headings appear bold and larger than normal text.

Strong Tag(<strong>)

  • Emphasizes important content, typically displayed as bold text.
  • Adds semantic meaning, signaling to search engines and screen readers that the content is significant.
  • Use it when you want to draw attention to critical information, such as warnings, key points, or important terms.

Emphasis Tag (<em>)

  • Emphasizes text, usually displayed in italic.
  • Adds semantic meaning, signaling subtle emphasis (e.g., stress or tone) to screen readers and search engines.
  • Use it to indicate stress or focus on specific words within a sentence (e.g., book titles, quotes, or key phrases).

Address Content Tag (<address>)

  • Formats contact information, often displayed in italic.
  • Indicates that the enclosed text represents contact information or addresses.
  • Use it for contact details or author information in a structured and semantic way.

Preformatted Text Tag (<pre>)

  • Preserves both spaces and line breaks as they are written.
  • Displays text in a monospaced font (like a code editor).
  • Retains whitespace and line breaks exactly as written in the HTML file.
  • Commonly used for displaying code snippets and showing content where formatting (spacing, indents, and line breaks) matters.

Citation Tag (<cite>)

  • Marks text as a citation, often rendered in italic.
  • Indicates a reference to a source, such as a book, article, or author.
